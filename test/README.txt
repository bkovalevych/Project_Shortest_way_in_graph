Есть всего три типа тестов. 
1 - где все данные корректные (нету лишних букв, только числа, которые не нарушают алгоритм);
2 - где данные корректные, но есть лишние буквы, которые могут быть отдельно от числа, 
либо же вместе с ним (Но при этом стоит учитывать, что числа изымаются изнутри букв);
3 - где есть случай 2, а еще присутствуют данные, которые нарушают работу алгоритма 
(наличие циклов, выход за пределы вершин и т. д.), при нахождении некорректной пары значений 
программа их игнорирует и считывает данные дальше. Но тогда возможна такая ситуация, когда данных 
для считывания не хватает, тогда алгоритм не будет выполняться (ибо это невозможно) и выведет
в res.txt указание на это.
